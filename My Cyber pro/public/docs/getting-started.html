Getting Started Guide

Everything you need to know to start using the Cyber Lab Platform

üöÄ Quick Start

Follow these steps to get up and running:

  1. Ensure Docker is running - Check that Docker Desktop is started
  2. Log in - Use the default credentials (admin / admin123)
  3. Choose a lab - Navigate to the Labs page
  4. Start the lab - Click the "Start Lab" button
  5. Wait for initialization - Labs typically start in 20-30 seconds
  6. Access the lab - Use the provided port to access lab services

üéÆ Using Labs

Starting a Lab

Click the "Start Lab" button. The platform will:

  • Pull required Docker images (first time only)
  • Create isolated network
  • Start all lab containers
  • Run health checks
  • Update status to "Running"

Accessing Lab Services

Once a lab is running, access it via:

http://localhost:[PORT]

The port number is displayed on the lab card.

Stopping a Lab

Click the "Stop" button to gracefully shut down the lab. All containers and networks will be cleaned up.

Restarting a Lab

If a lab encounters issues, use the "Restart" button for automatic recovery.

üîß Troubleshooting

Docker Not Running

Symptom: Red "Docker Not Running" badge
Solution: Start Docker Desktop and wait for it to fully initialize

Lab Stuck in "Starting"

Symptom: Lab shows "Starting" for more than 2 minutes
Solution: Click "Restart" or check Docker Desktop for errors

Port Already in Use

Symptom: Lab fails to start with port conflict
Solution: Stop other services using the same port or modify lab configuration

Lab Auto-Recovery

The platform includes automatic health monitoring. If a container crashes, it will attempt to restart automatically.
